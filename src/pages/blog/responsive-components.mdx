Building software interfaces requires contemplating multiple devices, while an interface could look and work great on desktop some of its components might not be the best choice for mobile or vice versa.

<DialogDemo client:load />

> Dialogs are the go to way to interrupt users to confirm an action, but they feel out of place on mobile.

And yes, you can always make use of a `useMobile` hook and render a different component according to screen size and you will get tired of doing that over and over again.

```tsx
import { useMobile } from '@/hooks/useMobile';
import { Dialog } from '@/components/ui/dialog';
import { Drawer } from '@/components/ui/drawer';

export const ConfirmationDialog = ({ isOpen, onClose, ...props }) => {
  const isMobile = useMobile();

  if (isMobile) {
    return <Drawer open={isOpen} onOpenChange={onClose} {...props} />;
  }

  return <Dialog open={isOpen} onOpenChange={onClose} {...props} />;
};
```
